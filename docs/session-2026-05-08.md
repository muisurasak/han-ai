## Flow ที่ทำงาน (Phase 1-2)

```
han init
  → interactive wizard
  → บันทึก ~/.han/config.json
  → บันทึก ~/.han/projects.json (ผ่าน UI ทีหลัง)

han start
  → อ่าน config
  → connect Redis (localhost:6379 dev / Upstash prod)
  → register machine ใน Redis HSET (heartbeat ทุก 30s)
  → polling loop:
      ทุก 30s (adaptive: ขยายสูงสุด 120s ถ้าไม่มีงาน)
        → query Notion DB: status == Approve
        → filter: assigned_to == me || null
        → filter: type ใน accept_types
        → filter: retry_count < 3
        → atomic claim via Redis SETNX (TTL 300s)
        → update Notion status → In-Progress
        → executeTask(task)
            ├── dev → dev-executor (Phase 4)
            └── doc/sheet/slide → not implemented yet
        → success: Notion status → Done + output_url
        → fail: retry_count++ → Approve (หรือ Failed ถ้า >= 3)
        → release Redis lock

han status
  → อ่าน Redis HSET
  → แสดง machine ทั้งหมด + online/offline + current task
```
